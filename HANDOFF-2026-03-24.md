## Branch
- `wip/ios-scroll-handoff-2026-03-24`

## Current objective
Stabilize iPhone Safari / PWA terminal behavior for:
1. reliable scrolling (keyboard hidden + keyboard shown)
2. no fake visual jumps
3. no endless `Connecting...`

## Key finding from uploaded debug log
Analyzed file:
- `files/tmp/gesture-debug-1774309316126.json`

Observed:
- gestures transition into `scroll`
- but terminal scroll state does not move:
  - `xtermViewportScrollTop = 0`
  - `termViewportY = 0`
  - `termBaseY = 0`
- only `visualViewport` changes (`vvTop`, `vvH`), matching the “screen jumps but content doesn't scroll” symptom.

Important interpretation:
- In that session, `termBaseY=0` means there was effectively no scrollback to scroll.
- Need logs with real long output before judging scroll pipeline.

### P1: implement iOS delegated scroll behind feature flag
- Add `iosScrollModel` flag (`native` / `arbiter`), default to `native`.
- For iOS flag path:
  - detect scroll gesture in wrapper
  - map dy to lines and call `term.scrollToLine(...)`
  - keep pinch as separate branch
  - suppress focus during/after scroll
- Keep non-iOS behavior unchanged.

### P1 status update (implemented on this branch)
- `client/app.js` now supports `iosScrollModel`:
  - query param: `?iosScrollModel=native|arbiter`
  - persisted key: `localStorage['ios_scroll_model']`
  - default: `native`
- Added iOS detection (incl. iPadOS-as-Mac touch-point case).
- Added delegated iOS scroll path (`arbiter` mode only):
  - converts touch `dy` to terminal lines using measured row height
  - applies scroll via `term.scrollToLine(...)` (fallback `term.scrollLines(...)`)
  - keeps pinch path separate
  - suppresses focus while scrolling and shortly after touch end
  - uses `preventDefault()` only in `iOS + arbiter` path
- Non-iOS and `native` behavior are unchanged.
